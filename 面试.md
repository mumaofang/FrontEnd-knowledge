1. BOM 和 DOM 底层逻辑
   BOM（浏览器对象模型）
   定义：BOM 提供与浏览器窗口交互的接口，用于管理浏览器行为（如导航、窗口尺寸、历史记录等）。
   BOM 的核心对象（如 window、location、navigator、history、screen）由浏览器内核（如 Chromium 的 Blink、Firefox 的 Gecko）实现
   事件循环与异步机制：BOM 的异步 API（如 setTimeout、XMLHttpRequest）依赖浏览器的事件循环。JavaScript 调用这些 API 时，浏览器内核会将任务推入任务队列，由事件循环调度执行
   DOM（文档对象模型）
   定义：DOM 是 HTML/XML 文档的树形表示，提供对文档结构和内容的编程接口。
   解析与构建：浏览器渲染引擎（如 Blink、WebKit）解析 HTML 时，逐行生成 DOM 树。每个 HTML 元素对应一个 DOM 节点，节点间形成父子关系。
       关键过程：词法分析 → 构建 DOM 节点 → 合并为树结构。 
   与渲染引擎的交互：DOM 修改（如 appendChild、style.color = 'red'）会触发渲染引擎的 重排（Reflow） 或 重绘（Repaint），影响性能。
   事件系统：DOM 事件（如点击、键盘输入）基于事件冒泡/捕获机制，由浏览器内核捕获原生事件并封装为 JavaScript 事件对象，通过事件队列传递。

3. ref 和 reactive
ref: 
    用于定义单个响应式数据，返回一个带有 .value 属性的对象。
    适合基本数据类型（如 string、number、boolean 等）。
    内部会将值包装成一个对象，通过 .value 访问和修改。
    如果传入对象或数组，会自动调用 reactive 转换。
reactive: 
    用于定义复杂数据结构的响应式对象（如对象、数组、Map、Set 等）。
    使用 Proxy 深度代理，内部自动递归处理。
    直接通过属性访问即可触发依赖收集。

‌赋值方式‌	   必须通过 .value 修改（如 count.value = 10）	            直接修改属性（如 state.count = 10）

4. 观察者模式和订阅发布模式的区别 --- 观察者模式是“主动通知”，发布-订阅模式是“通过中介传递消息”。

特性	    观察者模式	                       发布-订阅模式
通信方式	直接通信	                       通过消息代理间接通信
耦合度	    紧密耦合（Subject知道所有Observer）	松散耦合（发布者和订阅者不知道彼此）
关系	    一对一或一对多	                   多对多
触发时机	同步调用	                       通常是异步（通过消息队列）
错误处理	一个Observer出错可能影响整体	    更好的容错性

观察者模式：
Subject ──直接调用──> Observer1
          └─直接调用──> Observer2

发布-订阅模式：
Publisher ──发布事件──> Event Bus ──分发事件──> Subscriber1
                          └─分发事件──> Subscriber2
