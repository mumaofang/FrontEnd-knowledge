# 前端架构
前端架构是一个体现资深前端工程师综合能力的领域。<br/>
它不仅涉及技术选型，更关乎**工程化、可维护性、可扩展性、团队协作和业务支撑**。<br/>
这通常是中高级及以上岗位面试的核心。
# 核心理念 架构解决什么问题
前端架构的目标是在应用复杂度增长时，依然能保持：

- 开发高效： 新人容易上手，团队协作顺畅。

- 代码健康： 易读、易测试、易维护、低耦合。

- 体验卓越： 高性能、高可用、良好的用户体验。

- 稳定可靠： 线上监控、错误捕获、快速回滚。

# STAR-R 思维
- S（Situation）： 当时的业务背景和技术背景是什么？（例如：项目快速增长，多个团队并行开发，技术栈不统一）

- T（Task）： 需要解决的架构问题是什么？（例如：需要解耦、独立部署、提升协作效率）

- A（Action）： （这是重点） 你采取了什么行动？

 - 调研与选型： 对比了哪些方案（微前端方案A/B/C）？权衡标准是什么（团队熟悉度、社区生态、长期维护）？

 - 设计与实施： 设计了怎样的顶层架构图？制定了哪些规范（代码规范、提交规范、接口规范）？解决了哪些关键技术难点（样式隔离、状态共享）？

 - 协作与推进： 如何与团队沟通并落地？

- R（Result）： 带来了什么量化结果？（例如：构建时间减少30%，团队交付效率提升，应用加载性能提升）

- R（Reflection）： 有什么反思？如果可以重来，会做什么改进？

# 2. 阐述技术选型：
    框架： 选择 React 或 Vue（基于团队技术栈）。
    考虑使用 TypeScript 保障代码质量。
        react: 函数式编程，不可变数据
        vue:   渐进式框架，数据响应式
        typescript: 保障代码质量

    构建工具：rollup/tsup/vite/webpack
        Vite（开发体验）或 Webpack（生态成熟）。
        vite: 启动服务-》加载模块 -》 按需编译返回
        webpack: 源码 -》 依赖分析 -》 加载全部依赖 -》 打包 -》 启动服务
