# 关键点：
HTTPS = HTTP + SSL/TLS。
HTTP 负责内容（应用层）。
SSL/TLS 负责安全（表示层/会话层），它像是一个“安全套件”，包裹在 HTTP 外面。
它们共同运行在 TCP（传输层）提供的可靠连接之上。

# 三次握手
客户端发送syn -> 服务端接受syn并且返回syn-ack -> 客服端接受syn-ack并且向服务端发送ack -> 服务端接受ack


# 当你在浏览器输入 https://www.example.com 时：
浏览器（客户端）首先会与服务器的 443 端口 进行 TCP 三次握手，建立一条可靠的 TCP 连接。
TCP 连接建立成功后（进入 ESTABLISHED 状态），才会在这个连接之上，开始进行 TLS/SSL 握手（交换密钥、验证证书等）。
TLS 安全通道建立后，才开始传输加密的 HTTP 数据。
# 简单来说：TCP 三次握手负责“铺路”（建立可靠传输通道），TLS 握手负责“上锁”（建立安全加密通道），最后才是 HTTP 数据传输。
